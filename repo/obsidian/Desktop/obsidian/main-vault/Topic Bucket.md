### **Bucket 1: Physical & Network Standards (The "How it Connects" Bucket)**

**Concept:** Technologies dealing with radio waves, physical cables, hardware-level transmission, and the immediate data-link layer. If you can physically touch it, or if it involves an antenna, it likely goes here.

- **LTE / 3G / 4G / 5G:** Cellular telecommunication standards.
    
- **IEEE 802.3 (Ethernet) & 802.11 (Wi-Fi):** Physical LAN and WLAN standards.
    
- **MIMO (Multiple Input Multiple Output):** Physical antenna technology.
    
- **MAC (Media Access Control):** The hardware address burned into a network card (Layer 2).
    
- **CSMA/CD & CSMA/CA:** Hardware collision detection/avoidance rules for Ethernet and Wi-Fi.
    
- **CDMA / GSM:** Older physical cellular radio transmission standards.
    
- **ADSL (Asymmetric Digital Subscriber Line):** Physical transmission of data over copper telephone lines.
    
- **ISDN (Integrated Services Digital Network):** Early physical standard for simultaneous digital transmission of voice and data.
    
- **RFID / NFC (Radio Frequency Identification / Near Field Communication):** Short-range physical wireless data transfer.
    

### **Bucket 2: Software Protocols & Logic (The "How it Talks" Bucket)**

**Concept:** The invisible software rules for routing, formatting, and exchanging data _after_ a physical connection is established. These operate in the middle layers of the OSI model.

- **IP (Internet Protocol):** The logical addressing system for routing packets (IPv4, IPv6).
    
- **TCP / UDP (Transmission Control Protocol / User Datagram Protocol):** Rules for delivering packets (TCP guarantees delivery; UDP is fast but unguaranteed).
    
- **HTTP / HTTPS (Hypertext Transfer Protocol):** Rules for transmitting web page data.
    
- **DNS (Domain Name System):** The "phonebook" protocol translating URLs (like https://www.google.com/search?q=google.com) into IP addresses.
    
- **DHCP (Dynamic Host Configuration Protocol):** The protocol that automatically assigns IP addresses to devices joining a network.
    
- **ARP (Address Resolution Protocol):** The bridge protocol that links a logical IP address to a physical MAC address.
    
- **SMTP / POP3 / IMAP:** The protocols specifically for sending (SMTP) and receiving/syncing (POP3/IMAP) emails.
    
- **FTP / SFTP (File Transfer Protocol):** Rules specifically for transferring bulk files between a client and server.
    
- **VoIP (Voice over IP):** The methodology for digitizing and sending voice traffic over internet protocols.
    

### **Bucket 3: Management & Enterprise Tools (The "How IT Controls It" Bucket)**

**Concept:** High-level software, policies, or architectures used by system administrators to manage devices, users, security, or business operations.

- **MDM (Mobile Device Management):** Admin software to control employee phones.
    
- **SNMP (Simple Network Management Protocol):** Used by admins to monitor the health of network routers and switches (though a protocol, its _purpose_ is management).
    
- **Active Directory / LDAP:** Centralized database used by IT to manage user passwords and computer permissions in a Windows environment.
    
- **IAM (Identity and Access Management):** Enterprise frameworks for controlling who has access to what systems.
    
- **SSO (Single Sign-On):** An authentication scheme allowing a user to log in once to access multiple independent systems.
    
- **ERP (Enterprise Resource Planning):** Massive business management software (like SAP or Oracle) managing supply chain, HR, and accounting.
    
- **CRM (Customer Relationship Management):** Software (like Salesforce) used to manage customer data and sales pipelines.
    
- **IDS / IPS (Intrusion Detection/Prevention System):** Security administration tools that monitor network traffic for malicious activity.
    
- **SLA (Service Level Agreement):** Not software, but a business management policy defining guaranteed server uptime for clients.
